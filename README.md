# پنل اطلاع‌رسانی HR ایران‌هتل

یک پنل RTL فارسی برای مدیریت ارتباط منابع انسانی با کاندیداهای مصاحبه، همراه با اتصال Server-side به SMS.ir.

## امکانات نسخه فعلی

- داشبورد مدیریتی برای وضعیت جذب و مصاحبه‌ها
- بانک کاندیداها با ثبت موبایل، موقعیت شغلی، منبع جذب، شهر، مرحله جذب، امتیاز و یادداشت HR
- تقویم مصاحبه‌ها و تغییر سریع مرحله کاندیدا
- مرکز ارسال پیامک تکی/گروهی
- زمان‌بندی پیامک برای ۲۴ ساعت، ۳ ساعت و ۱ ساعت قبل از مصاحبه
- قالب‌های آماده برای دعوت به مصاحبه، یادآوری، تغییر زمان، نتیجه مثبت، عدم همکاری محترمانه و درخواست مدارک
- متغیرهای قالب مثل `{candidateName}`، `{position}`، `{interviewDate}`، `{interviewTime}`، `{location}` و `{signature}`
- ثبت لاگ کامل ارسال‌ها و خروجی CSV
- ورود گروهی کاندیداها با فرمت CSV ساده
- حالت تستی Dry Run برای تست بدون ارسال پیامک واقعی
- اتصال امن پیامک از API Route سرور؛ API Key داخل مرورگر نمایش داده نمی‌شود

## اجرای محلی

```bash
npm install
npm run dev
```

سپس آدرس زیر را باز کنید:

```bash
http://localhost:3000
```

## تنظیم SMS.ir

در روت پروژه فایل `.env.local` بسازید:

```env
SMS_IR_API_KEY=YOUR_API_KEY
SMS_IR_LINE_NUMBER=3000XXXXXXXXXX
SMS_IR_BASE_URL=https://api.sms.ir
SMS_DRY_RUN=false
```

برای تست بدون ارسال واقعی:

```env
SMS_DRY_RUN=true
```

## Deploy روی Vercel

1. پروژه را داخل GitHub بگذارید.
2. در Vercel گزینه Import Project را بزنید.
3. Environment Variables را مطابق `.env.example` ثبت کنید.
4. Build Command همان `npm run build` باشد.
5. بعد از Deploy از مسیر `/api/sms/health` وضعیت تنظیمات پیامک را بررسی کنید.

## نکته مهم درباره دیتابیس

این نسخه برای تحویل سریع و تست عملیاتی HR، داده‌ها را در `localStorage` مرورگر ذخیره می‌کند. برای نسخه سازمانی پیشنهاد می‌شود این موارد اضافه شود:

- PostgreSQL یا SQL Server برای ذخیره کاندیداها و لاگ‌ها
- ورود کاربران HR با نقش‌ها و سطح دسترسی
- Cron Job برای بررسی یادآوری‌ها و ارسال خودکار از سمت سرور
- اتصال به ATS یا فرم استخدامی سایت
- گزارش تحلیلی نرخ تبدیل هر موقعیت شغلی و هر منبع جذب

در همین نسخه، پیامک زمان‌بندی‌شده با ارسال `SendDateTime` به SMS.ir انجام می‌شود؛ بنابراین برای یادآوری‌های مصاحبه نیازی نیست کاربر HR در زمان ارسال آنلاین باشد.
